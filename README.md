## Обзор проекта

Проект реализует веб-сервис для визуализации глоссария терминов анализа поведения пользователей в виде интерактивного семантического графа. Система состоит из:

- **gRPC сервера** на Python для обработки запросов к глоссарию
- **REST API** для совместимости с фронтендом
- **Веб-интерфейса** с интерактивной визуализацией графа на D3.js
- **Docker контейнеров** для легкого развертывания

## Технологический стек

### Бэкенд:
- **gRPC** - высокопроизводительный RPC фреймворк
- **Protocol Buffers** - бинарный формат сериализации
- **Python 3.9** - язык серверной части
- **Flask** - REST API слой

### Фронтенд:
- **D3.js** - библиотека для визуализации графов
- **Vanilla JavaScript** - без фреймворков для простоты
- **CSS3** - современная стилизация

### Инфраструктура:
- **Docker** - контейнеризация приложений
- **Docker Compose** - оркестрация контейнеров

Использование системы

Веб-интерфейс

Откройте браузер и перейдите по адресу: http://localhost:8080
Основные функции:

Интерактивный граф: Кликните на любой узел для выделения
Поиск: Введите термин в поисковую строку
Информационная панель: Просмотр определений и связей
Сброс: Кнопка "Сбросить выделение" возвращает граф в исходное состояние
Визуальные элементы:

Синие узлы - термины
Красный узел - выбранный термин
Желтые узлы - результаты поиска
Серые линии - связи между терминами
API Endpoints

REST API:

GET /api/terms - получить все термины
GET /api/graph - получить данные для графа
gRPC методы:

GetAllTerms(Empty) returns TermList
GetTerm(TermId) returns Term
GetGraph(Empty) returns Graph
Анализ контейнеризации

Выбор Docker

Docker выбран по следующим причинам:

Стандартизация: Де-факто стандарт в индустрии
Портативность: Гарантированная работа на любой платформе
Изоляция: Каждый сервис в отдельном контейнере
Экосистема: Богатый набор инструментов и библиотек
Легкость развертывания: Простая установка и настройка
Отечественные альтернативы

1. OpenVZ/Virtuozzo

Плюсы: Российская разработка, высокая производительность
Минусы: Требует ядра с патчами, менее популярна
2. CRI-O

Плюсы: Совместима с Kubernetes, легковесная
Минусы: Меньше инструментов для разработки
3. LXC/LXD

Плюсы: Нативная поддержка в Linux, хорошая изоляция
Минусы: Сложнее в настройке для production
Преимущества выбранного решения

Быстрое развертывание: docker-compose up -d
Масштабируемость: Легко добавить новые сервисы
Воспроизводимость: Идентичная среда на всех машинах
Миграция: Простой перенос на другие платформы
